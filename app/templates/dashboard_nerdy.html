Source

Jigsaw / Kaggle

Models are trained on the Jigsaw Toxic Comment Classification Challenge dataset (Google, 2018), comprising English Wikipedia talk-page comments. The original dataset carries six fine-grained labels (toxic, severe_toxic, obscene, threat, insult, identity_hate); we collapse these into a single binary label — toxic (any label present) vs. non-toxic.

The strong class imbalance (~10 % positive) is handled by stratified train/test splitting. Threshold tuning on the validation set helps balance precision and recall for downstream moderation use-cases.

Feature Categories

Category Key Features Description
Slur & Lexicon slur_lexicon_hits, profanity_count, toxic_terms, lexicon_score Count of matches against curated toxicity and profanity word-lists. High hit counts are strongly predictive across all models.
Text Statistics caps_ratio, sentence_length Stylistic signals: proportion of capital letters (ALL CAPS correlates with aggression) and comment length (very short or very long comments skew toxic).
Sentiment neg_sentiment, sentiment_ratio Negative-affect score from a pre-trained sentiment lexicon. Complements bag-of-words by capturing hostile tone even when specific slurs are absent.
Semantic / Embedding context_score, intensity_measure, word_embeddings Distributed word-vector features encoding semantic similarity. Used primarily by the SVM and contribute to the ensemble.
