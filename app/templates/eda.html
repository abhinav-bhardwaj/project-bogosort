Feature Catalog

What this shows: All 32 hand-engineered features used for toxicity classification — their exact code names, data types, and what signal each captures. These features are computed from raw comment text at both training and inference time.

Top Features by Correlation

What this shows: Features ranked by their correlation with the target variable (toxicity). Stronger correlations indicate better predictive power.

Feature Means: Toxic vs Non-Toxic

Average value of each feature for toxic vs non-toxic comments. Search or sort by difference to find the most discriminative signals.

Feature Occurrence Analysis

How frequently each feature is non-zero, split by class. The gap column shows where toxic comments activate a feature more or less than non-toxic ones.

Identity Risk Analysis

What this shows: How toxicity rates differ for comments that mention specific identity groups — race, gender, sexuality, religion, disability, and nationality. A relative risk > 1 means comments mentioning that group are more likely to be labelled toxic in the training data, which can introduce model bias in production.

Feature Distribution Explorer

What this shows: The actual value distribution for each engineered feature, split by class. Select a feature to see how its values differ between toxic and non-toxic comments. Counts are shown as a percentage of each class so the heavily-imbalanced classes are comparable.

Feature Correlation Heatmap

What this shows: Pearson correlation between all 32 engineered features. Values near +1 (red) mean the two features move together; near −1 (blue) they move oppositely; near 0 (white) they are independent. Strongly correlated feature pairs may carry redundant information for the classifier.
