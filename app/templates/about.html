About the Project

ThermoTox is an interactive web application for detecting toxic content in Wikipedia talk page discussions. It was built in Spring 2026 by students at the Hertie School in Berlin as part of the Data Structures & Algorithms course in the Master of Data Science for Public Policy programme.

Background

Wikipedia is one of the largest collaborative knowledge-bases in the world. Behind every article is a talk page where editors discuss changes, flag disputes, and coordinate content decisions. These discussions can be constructive — but they can also turn toxic, discouraging participation and harming the volunteer community that keeps Wikipedia running.

This project explores whether machine learning can help surface toxic comments automatically, giving moderators a tool that scales with the enormous volume of talk page activity.

Dataset

Our models are trained on the Jigsaw Toxic Comment Classification Challenge dataset, published by Google's Jigsaw unit. It contains roughly 160,000 Wikipedia talk page comments labelled for toxicity, severe toxicity, obscenity, threats, insults, and identity-based attacks.

For this application we treat the task as binary classification (toxic vs. non-toxic), using the toxic label as the positive class.

Models

We trained and evaluated several classical machine learning classifiers using TF-IDF text vectorisation. All models are compared on accuracy, precision, recall, F1-score, and AUC on a held-out test set. You can inspect per-model metrics on the Nerdy Dashboard.

Model Notes
Lasso Logistic Regression L1-regularised; sparse, interpretable weights
Ridge Logistic Regression L2-regularised; strong baseline
Linear SVM High-dimensional text classification workhorse
Random Forest Random ensemble of decision trees
Ensemble Ensemble of Lasso, Ridge, and Random Forest

Technical Stack

  • Backend: Python · Flask · SQLite (via SQLAlchemy)
  • ML: scikit-learn · NumPy · pandas
  • Data fetching: Manual Talk Fetch Implementation with MediaWiki REST API
  • Frontend: Vanilla HTML/CSS/JS (Wikipedia-inspired design)
  • Testing: pytest · coverage
